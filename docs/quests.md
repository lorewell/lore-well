# 任务与任务相关对话手册

> 本文以当前代码实现为准，不按设计意图脑补。
> 任务是否激活、目标何时勾选、奖励何时发放，全部以下列文件为准：
>
> - `src/data/quests.ts`
> - `src/data/npcs.ts`
> - `src/data/locations.ts`
> - `src/data/shops.ts`
> - `src/data/enemies.ts`
> - `src/store/gameStore.ts`

---

## 先读这几条规则

### 1. 开启 NPC 对话就会触发 `talk_npc`

任务目标里的 `talk_npc` 不是“选到某一句台词”才完成，而是只要打开与该 NPC 的对话框就会触发。

### 2. 所有目标都是自动勾选

系统会在以下事件发生时自动检查任务目标：

- 获得物品：`have_item`
- 击败敌人：`defeat_enemy`
- 前往大地点：`visit_location`
- 打开 NPC 对话：`talk_npc`

### 3. 所有目标完成后，奖励自动发放

不需要手动回报任务，也不需要额外点“提交”。

### 4. 每个对话节点都会额外出现通用收尾选项

除数据里写死的选项外，`DialogBox` 会始终追加一条：

- `▶ 没事了。`

下文的分支树默认不重复写这一条，但它始终存在。

### 5. `shop` 不是普通对话分支，而是特殊动作

当某个对话选项的 `next` 是 `shop` 时：

- 会直接打开该 NPC 绑定的商店
- 会立即关闭当前对话框

### 6. 客栈老板娘的 `rest` 也是特殊动作

与玛格对话时，如果进入 `rest` 分支，会立即回复 HP / MP，然后进入对应的文本节点。

## 全任务总览

| 任务 | ID | 运行时激活方式 | 自动完成条件 | 奖励 |
|---|---|---|---|---|
| 初来乍到 | `quest_arrive` | 新开档后立即激活 | 与村长格雷对话、与艾尔文对话 | 80 EXP |
| 补给短缺 | `quest_supplies` | 新开档时已是进行中 | 获得 `health_potion` | 50 EXP |
| 长老的委托 | `quest_elder` | 第一次打开艾尔文对话时激活 | 第一步会在同一次对话中自动完成，之后获得 `ancient_key` 并前往 `temple_ruins` | 200 EXP + 100 G |
| 森林的威胁 | `quest_forest` | 第一次打开艾尔文对话时激活 | 击败 `goblin`，击败 `forest_wolf` | 150 EXP + 60 G |
| 铁匠的请托 | `quest_blacksmith` | 第一次打开托尔对话时激活 | 第二步会在同一次对话中自动完成；在任务激活后再获得 `iron_ore` 即可完成 | 120 EXP + 80 G + 1 把钢剑 |

## 线性任务流程

下面按玩家正常推进顺序，把任务、地点和人物对话分支串起来写清楚。

## 0. 开局：小莉娜序章

### 运行时会发生什么

调用 `startNewGame()` 时，系统会同时做这几件事：

- 把当前对话设置为 `lina_prologue / greeting`
- 立即激活 `quest_arrive`
- 保持 `quest_supplies` 为进行中（它在初始任务表里本来就是 `active`）

### 当前可见任务

- `quest_arrive`：进行中
- `quest_supplies`：进行中

### 小莉娜序章对话树

```text
greeting
├─ “……我在哪里？” -> where
│  └─ “艾尔文？那位老先生现在在哪里？” -> find_elder
│     └─ “好，我去找他。谢谢你，莉娜。” -> 结束
└─ “（努力回忆，却什么都想不起来。）” -> confused
   └─ “艾尔文……那位老先生现在在哪里？” -> find_elder
      └─ “好，我去找他。谢谢你，莉娜。” -> 结束
```

### 这段对话的作用

- 只负责开场叙事
- 不会直接完成任何任务目标
- 会把你引向村长之屋和艾尔文的家

## 1. 去村长之屋：推进“初来乍到”第一步

### 实际路径

从落瀑村起点 `中心广场` 出发：

1. 先往北到 `艾尔文的家`
2. 再往西到 `村长之屋`

说明：小莉娜的台词写的是“从中心广场往西北走”，但按当前子地图实现，实际节点路径是“先北、再西”。

### 触发点

只要打开村长格雷的对话框，就会触发：

- `quest_arrive.objective.visit_chief` 完成

### 村长格雷对话树

```text
greeting
├─ “村子最近出了什么事？” -> trouble
│  ├─ “我会调查这件事的。” -> grateful -> 结束
│  └─ “听起来很危险……” -> 结束
├─ “我能帮上什么忙吗？” -> help -> 结束
└─ “好，我去找艾尔文。谢谢村长。” -> chief_farewell -> 结束
```

### 这段对话的剧情作用

- 村长告诉你：艾尔文已经回自己家了
- 给出三个线索方向：
  - 村里最近不太平
  - 艾尔文知道得更多
  - 托尔与玛格也值得去找

### 这一步后，任务状态

- `quest_arrive`
  - `前往村长之屋，打听艾尔文的下落`：已完成
  - `前往艾尔文的家，与他交谈`：未完成

---

## 2. 去艾尔文家：主线与森林支线同时解锁

### 实际路径

从村长之屋往东返回艾尔文的家。

### 触发点

只要第一次打开艾尔文的对话框，系统就会立刻做四件事：

1. 激活 `quest_elder`
2. 激活 `quest_forest`
3. 完成 `quest_arrive.objective.talk_elder_first`
4. 完成 `quest_elder.objective.talk_elder`

如果你此前已经和村长说过话，那么在这一次打开艾尔文对话时：

- `quest_arrive` 会立刻自动完成
- 奖励 `80 EXP` 会立刻发放

### 艾尔文对话树

```text
greeting
├─ “我的记忆全部消失了。” -> memory_lost
│  └─ “村子现在还安全吗？” -> village_danger
│     ├─ “告诉我更多关于神殿的事。” -> more -> 结束
│     └─ “我还没准备好。” -> 结束
├─ “村子现在还安全吗？” -> village_danger
│  ├─ “告诉我更多关于神殿的事。” -> more -> 结束
│  └─ “我还没准备好。” -> 结束
└─ “是你救了我？谢谢你。” -> thanks -> 结束
```

### 重要实现说明

#### 1. `quest_elder` 的第一步会立刻完成

它不是“听完某句台词才算”，而是对话框一打开就完成。

#### 2. `quest_forest` 也会在这里一起解锁

即使你没有点到“告诉我更多关于神殿的事”，只要打开艾尔文对话，森林支线也会变成进行中。

#### 3. 艾尔文不会真的把钥匙给你

`more` 节点的文本里写着：

> “带上这把古代钥匙，也许它能打开神殿的大门。”

但当前实现里：

- 不会通过对话直接发放 `ancient_key`
- 真正的钥匙必须去 `森林深处 / 石台遗址 / 石台上的匣子` 拾取

## 3. 可选先做：补给短缺

### 任务状态

`quest_supplies` 从新开档开始就是进行中。

### 完成条件

只要背包里第一次加入 `health_potion`，任务就会立刻自动完成并发放：

- `50 EXP`

### 当前版本的三种稳定完成方式

#### 方式 A：找老板娘玛格买药

地点：`暮光客栈`

对话树：

```text
greeting
├─ “我想休息（回复满血）” -> rest -> 结束
├─ “我想买些物资。” -> shop -> 打开客栈商店
├─ “最近有什么消息吗？” -> news
│  └─ “我知道了，谢谢。” -> 结束
└─ “不了，谢谢。” -> 结束
```

客栈商店里：

- `health_potion` 价格为 `30 G`

#### 方式 B：找杂货商梅娜买药

地点：`杂货铺`

对话树：

```text
greeting
├─ “让我看看你的货。” -> shop -> 打开杂货铺商店
├─ “村子里最近有什么动静？” -> gossip
│  └─ “我明白了，谢谢你。” -> 结束
└─ “不了，随便逛逛。” -> 结束
```

杂货铺商店里：

- `health_potion` 价格为 `25 G`

也就是说，按当前实现，梅娜是完成这个任务最便宜的购买路线。

#### 方式 C：去森林捡药

地点：`幽暗森林 / 林中空地 / 废弃的行囊`

这是一处一次性交互，拾取后会直接往背包里加 `health_potion`。

### 额外掉落来源

当前敌人掉落表里：

- `goblin` 有 `30%` 概率掉 `health_potion`

注意：

- 当前 `slime` 没有生命药水掉落
- 旧文档里写的“史莱姆掉药”已经与代码不一致

## 4. 做森林支线：森林的威胁

### 任务何时开始

第一次打开艾尔文对话后解锁。

### 完成条件

需要各完成一次：

1. 击败 `goblin`
2. 击败 `forest_wolf`

只要两者都发生过，任务会立刻自动完成，奖励：

- `150 EXP`
- `60 G`

### 推荐路线

从 `落瀑村 / 村南路口` 出村，进入 `幽暗森林`：

1. `森林入口`
   - `哥布林巡逻队`：可完成 `goblin`
2. `林中深处`
   - `森林狼群`：可完成 `forest_wolf`

也可以在更深的位置完成：

- `森林深处 / 深处入口 / 古老的狼灵`
- `森林深处 / 古树密处 / 狼灵首领`

这些敌人的 `targetId` 仍然是 `forest_wolf`，所以同样会计入任务。

### 注意

这个任务是纯自动结算：

- 不需要再回村找艾尔文
- 不需要额外汇报

## 5. 推进主线：长老的委托

### 任务何时开始

第一次打开艾尔文对话后解锁。

### 当前实现中的目标顺序

#### 第一步：与长老艾尔文交谈

这一项会在第一次打开艾尔文对话时立即完成。

#### 第二步：在森林深处找到古代钥匙

实际地点：

- `森林深处 / 石台遗址 / 石台上的匣子`

这会往背包里加入：

- `ancient_key`

任务目标会在物品入包的那一刻自动勾选。

#### 第三步：前往古代神殿遗迹

这里检测的是大地点：

- `visit_location: temple_ruins`

也就是说，只要你从 `村南路口` 进入 `古代神殿遗迹`，这一项就会立刻完成。

### 当前实现的完成时点

只要满足以下三件事：

1. 已经在对话里见过艾尔文
2. 已经拿到 `ancient_key`
3. 进入 `temple_ruins`

任务就会立刻自动完成，奖励：

- `200 EXP`
- `100 G`

### 当前版本不需要做的事

按代码实现，完成这个任务时不需要：

- 再回去找艾尔文
- 与神殿深处的 `Lore Well` 交互
- 使用钥匙开门

## 6. 做铁匠支线：铁匠的请托

这是当前实现里最需要按代码逻辑理解的任务。

### 任务何时开始

第一次打开铁匠托尔对话时解锁。

### 托尔对话树

```text
greeting
├─ “我想看看你的商品。” -> shop -> 打开铁匠铺商店
├─ “我有铁矿石。” -> has_ore -> 结束
└─ “随便看看。” -> 结束
```

### 重要实现说明

#### 1. 第二步会在第一次对话时自动完成

任务第二步写的是：

- `带铁矿石回去找铁匠托尔`

但当前实现里，任务激活和 `talk_npc(blacksmith)` 检查发生在同一次打开对话里，所以：

- 你第一次和托尔说话时
- 这一步就已经会被勾掉

#### 2. “我有铁矿石。” 这个选项没有库存校验

不管你背包里有没有 `iron_ore`，这个选项都会出现，并会进入 `has_ore` 文本。

#### 3. 真正卡任务的是第一步“获得铁矿石”

这一项只有在任务已经激活后，再次触发 `have_item(iron_ore)` 才会完成。

这意味着：

- 如果你在接任务之前就拿到了铁矿石
- 之后再去见托尔
- 系统不会补算“获得铁矿石”这一步

所以想稳定完成，请按下面顺序走。

### 当前版本的稳定完成顺序

#### 第一步：先找托尔说一次话

地点：`落瀑村 / 铁匠铺`

这会：

- 激活 `quest_blacksmith`
- 自动完成 `return_to_smith`

#### 第二步：在任务激活后，再获得一次 `iron_ore`

当前有三条有效来源：

1. `废弃矿洞 / 矿脉区 / 裸露的矿脉`
2. `cave_spider` 掉落，概率 `50%`
3. `grocer_shop` 直接购买，价格 `15 G`

### 当前实现的真实完成时点

只要在任务激活之后，`iron_ore` 被加入背包：

- 任务第一步完成
- 因为第二步早就完成了
- 所以整个任务会当场自动完成

当前版本不需要再回铁匠铺交任务。

### 奖励

- `120 EXP`
- `80 G`
- `steel_sword ×1`（显示名：`钢剑`）

### 最快完成法

按纯实现逻辑，最快流程是：

1. 先和托尔说话一次，激活任务
2. 立刻去梅娜的杂货铺买 `iron_ore`（15 G）
3. 任务当场自动完成

净收益是：

- 花费 `15 G`
- 获得 `80 G`
- 获得 `120 EXP`
- 获得 `钢剑 ×1`

## 容易写错的实现细节

### 1. `quest_arrive` 不是一直锁着

虽然数据表里写的是 `locked`，但新开档后会立刻调用：

- `activateQuest('quest_arrive')`

### 2. `quest_elder` 与 `quest_forest` 不是听完艾尔文整段故事才解锁

只要打开艾尔文对话框就解锁。

### 3. 艾尔文的“古代钥匙”只是台词，不是发奖

真正的 `ancient_key` 来源是：

- `森林深处 / 石台遗址 / 石台上的匣子`

另外，`quest_elder` 的任务描述里还残留了旧地名“晨曦村”，但当前地图、地点名和 NPC 文本使用的都是“落瀑村”。本文统一按当前可玩内容写成“落瀑村”。

### 4. `quest_blacksmith` 不是“拿矿石后回去交差”

按当前实现：

- 先对话就会完成“回去找铁匠”
- 后拿到矿石就会立刻完成整个任务

### 5. 如果你在接铁匠任务前就拿到了矿石，系统不会补算

因为 `have_item(iron_ore)` 只会在“获得物品的那一刻”检查一次。

所以稳定流程一定是：

1. 先和托尔说话
2. 再去获得矿石
